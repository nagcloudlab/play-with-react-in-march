why we need javascript on HTML-pages?


using browser's Apis, javascript can do any magic on the webpage, for example:

browser's Apis => w3c recommendations, implemented by browser vendors navivly ( c++ ),
abstracted as javascript objects and methods, in global-scope, accessible from any script on the page

e.g DOM,XHR/Fetch,Timers, etc.

using DOM API, javascript can:
-------------------------------
-> create/retrieve/update/delete HTML-elements
-> react to user events (click, hover, scroll, etc.)
-> change styles dynamically

using XHR/Fetch API, javascript can:
-------------------------------
-> make asynchronous requests to the server

using Timers API, javascript can:
-------------------------------
-> execute some actions on time intervals (setTimeout, setInterval)

complexities while working browser apis using plain-javascript ( your script)
------------------------------------------------------
1. browser compatibility issues ( different browsers may implement APIs differently, or not support some features at  all )
2. performance issues ( manipulating DOM can be expensive, especially if you have a lot of elements or complex interactions )
3. memory leaks ( if you create a lot of event listeners or DOM elements without proper cleanup, it can lead to memory leaks )
4. security issues ( if you don't properly sanitize user input or handle data, it can lead to security vulnerabilities like XSS attacks )
5. code maintainability ( as your codebase grows, it can become difficult to manage and maintain, especially if you have a lot of interdependent code )
